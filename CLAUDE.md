## Project Overview

Single-file Python utility that fetches a snapshot from a Reolink camera via its HTTP API and saves it to a NAS (QNAP via NFS). Designed to run as a cron job or Docker container.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Copy and configure before first run
cp config.example.ini config.ini

# Run the snapshot (requires config.ini and a reachable camera + NAS)
python snapshot.py

# Build Docker image locally
docker build -t timelapse .

# Run via Docker (one-off)
docker run --rm \
  -v /path/to/config.ini:/app/config.ini:ro \
  -v /mnt/qnap/timelapse:/mnt/qnap/timelapse \
  ghcr.io/mdormann/timelapse:latest
```

There is no test suite. Manual testing is done by running `python snapshot.py` directly.

## Architecture

Everything lives in `snapshot.py` (~90 lines). The flow is linear:

1. `load_config()` — reads `config.ini` (INI format via `configparser`); exits with code 1 if missing
2. `fetch_snapshot()` — calls the Reolink CGI API (`/cgi-bin/api.cgi?cmd=Snap`) with credentials and a random `rs` parameter (generated by `random_rs()`); validates the response is an image by checking `Content-Type`
3. `save_snapshot()` — creates a date-based subdirectory under the configured NAS path and writes the JPEG bytes with a timestamped filename
4. `main()` — wires the above together; exits with code 1 if the NAS path is unreachable

Config is resolved relative to `snapshot.py`'s own directory (`Path(__file__).parent / "config.ini"`), so Docker mounts must place `config.ini` at `/app/config.ini`.

## Configuration

`config.ini` (gitignored) is based on `config.example.ini`:

```ini
[camera]
host = 192.168.1.100
user = admin
password = DEIN_PASSWORT
channel = 0        # camera channel index
timeout = 15       # HTTP request timeout in seconds

[nas]
path = /mnt/qnap/timelapse
filename_prefix = snapshot
```

Output files land at `<nas.path>/<YYYY-MM-DD>/<filename_prefix>_<YYYY-MM-DD_HH-MM-SS>.jpg`.

## CI/CD

GitHub Actions (`.github/workflows/docker.yml`) builds a Docker image on every push to `main` and on PRs. It pushes to `ghcr.io/mdormann/timelapse` with the tags `latest` and `sha-<commit>` — but only on non-PR events (PRs trigger a build-only check). The registry is GitHub Container Registry (GHCR), authenticated via `GITHUB_TOKEN`.

## Conventions

- Log messages and error strings are in German.
- The script uses `sys.exit(1)` (not exceptions) for operational failures (missing config, unreachable NAS).
- No framework, no CLI argument parsing — configuration is entirely via `config.ini`.
- Python 3.8+ compatible syntax; the Dockerfile targets 3.12-slim.
